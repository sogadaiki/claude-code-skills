# YouTube Script: $ARGUMENTS

YouTube動画の台本を作成する。

## 必須参照

台本作成前に必ず読み込む:
- `~/.claude/knowledge/youtube-scriptwriting.md` — 台本作成ベストプラクティス
- `~/.claude/knowledge/scriptwriting-patterns.md` — 台本パターン集
- `~/.claude/knowledge/youtube-script-production.md` — 制作プロセス
- `~/.claude/knowledge/notebooklm-slide-designs.md` — NotebookLMスライドデザイン指示集（12種）
- `~/.claude/knowledge/short-video-buzz-playbook.md` — ショート動画（ショート動画時のみ）

## Multi-Pass Design

既存ワークフロー（Step 1-6）を多段階原則にマッピング。

| Pass | 内容 | 対応Step |
|------|------|----------|
| **Pass 1: 素材収集** | テーマ/データ読み込み、ナレッジ5ファイル参照、Web検索補足、チャンネル実績把握 | Step 1 |
| **Pass 2: 構成設計 + 自己批判** | 感情カーブ・オープンループ・スライド指示を設計。設計後に「冒頭15秒で離脱させないか？」「テンション3段階（低→中→高）になっているか？」「データ羅列になっていないか？」を自己批判。方針確認含む | Step 2-3 |
| **Pass 3: 生成** | 台本本文 + 付属コンテンツ（概要欄・タイトル・サムネイル方針）を執筆 | Step 4-5 |
| **Pass 4: 品質ゲート** | セルフレビュー表の全項目チェック | Step 6 |
| **Pass 5: 修正** | FAIL項目のみリライト。2回FAILでそのセクションをPass 2に戻して再構成 | Step 6後 |

### Pass 4 品質ゲート項目（台本固有）

| # | チェック項目 | FAIL基準 |
|---|------------|---------|
| 1 | フック(0-8秒) | 好奇心ギャップなし / 抽象的すぎる |
| 2 | OP/自己紹介 | 3秒超 |
| 3 | データ読み上げ | 年度ごと羅列（結論だけ言っていない） |
| 4 | オープンループ | 2-3分間隔で配置されていない |
| 5 | パターンインタラプト | 25-35秒ごとの指示がない |
| 6 | 感情カーブ | フラット区間が2分以上連続 |
| 7 | 当事者化 | 「あなた」への語りかけがない |
| 8 | 禁止表現 | 法務NGワード含有 |
| 9 | 図解指示 | 1sceneに図解2つ以上 / @秒数なし / カメラ復帰なし |

## 手順

### 1. 情報収集

以下を並列で実行:

**データ収集**
- 指定されたテーマ/Issue/データファイルを読み込む
- 関連するナレッジファイルを参照（プロダクト情報、事業情報等）
- 必要に応じてWeb検索で補足情報を取得

**チャンネル情報確認**
- 対象チャンネルの過去動画実績（視聴率、CTR等）があれば把握
- 前作・シリーズの流れを確認

### 2. 方針確認（AskUserQuestion）

以下をユーザーに確認:
- **台本構成の型**: 発見追体験（ミステリー）/ ストーリーテリング / リスト形式
- **想定尺**: 5-8分 / 10-12分 / 15分超
- **トーンの方向性**: データ重視（冷静）/ 感情重視（熱い）/ バランス
- **CTA先**: 自社サービス / チャンネル登録のみ / 外部リンク
- **法務上の注意点**: 社名を出すか / 禁止表現 / 免責の要否

### 3. 台本構成の設計

ナレッジの構成パターンに従い、以下を設計:

1. **感情カーブ**: 時間軸に沿った感情の設計図
2. **オープンループの配置**: セクション末尾の引きポイント
3. **パターンインタラプトの配置**: 25-35秒ごとの変化ポイント
4. **NotebookLMスライド指示**: 統一デザインスタイル選定 + 各スライドのTYPE/内容/個別指定
   - **指示は「初めて仕事を受けるデザイナー」に渡すレベルで書く**。見出しテキスト、表示する数値、グラフの軸・データポイント、コールアウト位置、出典テキスト、ボトムメッセージを全て日本語で正確に明記する
   - NotebookLMは文脈を全て読み込む力がある。細かいほど正確に再現する。曖昧だと文字やデータを適当に埋める
5. **サムネイル案 x 3**: NotebookLMで同時生成
6. **チャプター構成**: 概要欄に載せるタイムスタンプ

### 4. 台本本文の執筆

以下の要素を全て含む:

- **HOOK（0-8秒）**: 視覚的インパクト + 好奇心ギャップ
- **本編**: 選択した型に沿った構成
- **「あなたへ」セクション**: 視聴者を当事者化する語りかけ
- **CTA + 次回予告**: 具体的で刺激的な次回予告

各セクションに以下のメタ情報を付記:
- `[テロップ: ...]` — テロップ/テキストオーバーレイの指示
- `[図表N: 説明 @Ns]` — 図表の差し込み（**edit-video連携ルール必読**: `youtube-script-production.md` 図解指示セクション）
- `[画面: カメラに切り替え]` — 図表終了→カメラ復帰
- `[パターンインタラプト: ...]` — 編集指示
- `[効果音: ...]` — SE指示
- `[BGM: ...]` — BGMの雰囲気変更指示

**図解(図表)を入れる場合の必須ルール** (詳細は `youtube-script-production.md`):
1. 図解ごとにsceneを分割する（1つのsceneに図解は1つまで）
2. `@Ns` で表示秒数を明示する（テキストカード5-8s、グラフ10-15s）
3. 図解中の台詞はその図解の内容に限定する
4. `[画面: カメラに切り替え]` を省略しない
5. 図解直後の最初の台詞にユニークなキーワード（固有名詞・数値）を含める

### 5. 付属コンテンツの作成

台本と合わせて以下も生成:
- **概要欄テンプレート**: チャプター + UTMリンク + 免責 + ハッシュタグ
- **ピン留めコメント**: CTA + 質問歓迎
- **タイトル候補**: 3案（数字入り推奨）
- **サムネイル方針**: 色調、テキスト、表情の方向性

### 6. セルフレビュー

完成した台本を以下の観点でチェック:

| チェック項目 | 基準 |
|-------------|------|
| 最初の8秒 | 好奇心ギャップがあるか |
| OP/自己紹介 | 3秒以下 or 削除されているか |
| データの読み上げ | 年度ごとの羅列がないか（結論だけ言っているか） |
| オープンループ | 2-3分ごとに配置されているか |
| パターンインタラプト | 25-35秒ごとに指示があるか |
| 感情カーブ | フラットになっていないか |
| 「あなた」 | 視聴者の当事者化があるか |
| 禁止表現 | 法務NGワードが含まれていないか |
| CTA | 自然に配置されているか（中盤+ED） |
