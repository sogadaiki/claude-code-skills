# Plan First: $ARGUMENTS

**いきなりコードを書かない。まず設計図を完璧にする。**

## Phase 1: 要件整理 (20%)

- 何を作るのか明確化
- 成功条件の定義(完了の判定基準)
- 制約・前提条件の洗い出し
- 関連するナレッジ・既存実装の参照
- スキル設計時は `~/.claude/knowledge/claude-code-skill-patterns.md` を参照

## Phase 2: 設計 (50%)

- アーキテクチャ設計
- データフロー(入力→処理→出力)
- インターフェース定義(型、API契約)
- エッジケースの列挙
- テスト戦略
- 必要に応じてサブエージェントで調査

## Phase 2.5: Devil's Advocate 検証 (10%) -- 必須

**Phase 2の設計が完成したら、必ずDevil's Advocateモードで検証する。**

手順:
1. Phase 2の設計ドキュメントをレビュアー（サブエージェント or 自問）に渡す
2. 「Devil's Advocate」で全否定検証を実行
3. 出力: 致命的欠陥 + 対案
4. 対案を踏まえて設計を修正 or 「否定を上回る根拠」を明示して維持
5. 最終設計をユーザーに提示（「否定 → 対応」を併記）

**スキップ条件**: 以下の場合のみスキップ可
- 既存バグ修正で設計変更がない場合
- ユーザーが明示的にスキップを指示した場合

## Phase 3: 実装 (20%)

- 設計に沿ってステップバイステップで実装
- 各ステップで型チェック検証
- 問題があれば計画に戻る(**無理やり押し進めない**)

## 出力

1. 設計ドキュメント
2. 実装計画(ステップバイステップ、依存関係明記)
3. リスク・懸念事項リスト

**うまくいかない時は無理やり押し進めるな。Plan Modeに戻れ。**

### 使い方

```
/plan ユーザー認証機能の追加
/plan Stripe決済フローの実装
/plan リアルタイム通知機能
```
