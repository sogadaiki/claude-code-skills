# Morning Start

**設計原則**: 軽量に起動し、作業決定後にナレッジを読む。business.md / decisions.md は朝の段階では読まない。

## 0. 実行コンテキスト判定（最初に必ず実行）

```bash
git remote get-url origin 2>/dev/null
```

| 結果 | モード | スコープ |
|------|--------|----------|
| メインリポ（ops/管理リポ）を含む | **司令塔モード** | 全リポジトリ横断 |
| 上記以外のURL | **プロジェクトモード** | このリポのみ |
| コマンド失敗（remote無し） | **ローカルモード** | git status + restart.mdのみ |

## 1. 環境状態の確認

```bash
git fetch && git status
git log --oneline -5
```

### 司令塔モード限定: repo-status.md 自動生成

全リポの最終コミットを `~/.claude/knowledge/repo-status.md` に出力する。
これにより他リポのClaude Codeセッションからも全体状況を参照できる。

```bash
echo "# Repository Status (auto-generated by /morning)" > ~/.claude/knowledge/repo-status.md
echo "Updated: $(date '+%Y-%m-%d %H:%M')" >> ~/.claude/knowledge/repo-status.md
echo "" >> ~/.claude/knowledge/repo-status.md
for repo in ~/Desktop/development/*/; do
  name=$(basename "$repo")
  [ -d "$repo/.git" ] || continue
  last=$(git -C "$repo" log -1 --format="%h %s (%cr)" 2>/dev/null)
  echo "- **$name**: $last" >> ~/.claude/knowledge/repo-status.md
done
```

このファイルは他スキルから読み込まれるが、`/morning` 自体では表示に使わない（Issue一覧で十分）。

## 2. Issue状況

### 司令塔モード

以下のリポジトリのopen issuesを **並列で** 取得する:

```bash
# 自分のリポジトリに合わせて変更してください
gh issue list --repo <your-org>/<main-repo> --state open --limit 15
gh issue list --repo <your-org>/<project-a> --state open --limit 15
gh issue list --repo <your-org>/<project-b> --state open --limit 10
```

取得できないリポジトリはスキップ（`|| true`）。直近マージPRも簡潔に（`--limit 5`）。

### プロジェクトモード

このリポのみ: `gh issue list --state open --limit 15` + `gh pr list --state merged --limit 5`

### ローカルモード

スキップ。

## 3. 前回の続きを把握

- `restart.md` が存在すれば **最優先で表示**（削除しない）
- MEMORY.mdの「次回セッション最優先」セクションがあれば表示

## 4. 本日の作業提案

Issue + restart.md + MEMORY.md + _registry.yaml/project.yaml + コミット履歴 から優先順位を提案する。

### 司令塔モード — 優先順位ロジック（上から順に評価）:
1. RED **関係温度マップでREDのクライアント** → 対人フォローは最優先
2. `priority:high` ラベルのIssue
3. 前回の作業の続き（restart.md）
4. 期限が近いIssue
5. その他のopen Issue

- **どのリポの何番Issueか** を明示。開発作業なら「別ウィンドウで開いて作業」と案内
- REDクライアントが存在する場合、作業提案の冒頭に配置する

### プロジェクトモード
- このリポのIssueだけで提案
- セクション2.3でRED（15日超未接触）を検出した場合、対人フォローを作業提案の冒頭に配置

### 提案後

ユーザーが作業を選択したら:
1. `~/.claude/knowledge/INDEX.md` を読む
2. タスクのキーワードでクラスタを特定
3. クラスタのファイル群 + `also:` 先のクラスタのファイルを読み込み
4. `also:` の連鎖は **1段階まで**（2段階以上は追わない）
