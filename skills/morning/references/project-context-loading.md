# Step 2.3: プロジェクトコンテキスト（プロジェクトモード限定）

該当プロジェクトの `memory/*-architecture.md` セクション0を自動検出し、人間文脈を表示する。

## 検出ロジック

1. リポ名を取得:
```bash
basename "$(git remote get-url origin 2>/dev/null)" .git
```

2. リポ名からメインリポのmemory内の該当architecture.mdを探す:
   - パス: `~/.claude/projects/<project-memory-path>/memory/{slug}-architecture.md`
   - **マッピング**: リポ名と設計書slugの対応（必要な場合のみ。一致するならそのまま使用）
   - 該当ファイルが存在しなければ **スキップ**（自社プロダクトリポ等）

3. 存在する場合、**セクション0のみ** を読み、以下のフォーマットで表示:

```
## プロジェクトコンテキスト
- **キーパーソン**: 田中さん（代表、敬語、成果重視）
- **意思決定**: 田中さんが単独判断
- **地雷**: なし
- **成功**: 月5件の面談設定
```

## 地雷の常時表示

地雷が「特記なし」でなければ、**毎回**目立つ形で表示する:

```
WARNING 地雷: {地雷の内容をそのまま表示}
```

## 関係者アクション候補

セクション0の情報と `clients/{path}/` への最終コミット日を照合し、フォローが必要かを1行で表示:

```bash
git -C <ops-repo-path> log -1 --format="%ad" --date=short -- "clients/{path}/" 2>/dev/null
```

```
## 関係者アクション
- 田中さん: 3/8にレポート送付済み → 次回接触は週次MTG（予定通り）
```

## セクション0鮮度

最終更新が30日超なら「この情報は現在も正確ですか？」と1行確認する。
