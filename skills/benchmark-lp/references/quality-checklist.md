# Quality Checklist (Phase 5 詳細)

## 5.1 ビジュアル検証（Playwright）

```
スマホ（375px）とPC（1440px）のスクリーンショットを撮影し、
ベンチマークサイトと並べて視覚比較する。
```

Playwright MCPまたは `/webapp-testing` スキルを使用。

## 5.2 チェックリスト

| # | チェック項目 | 合格基準 |
|---|------------|---------|
| 1 | モバイル表示 | 375pxで崩れなし |
| 2 | PC表示 | 1440pxで崩れなし |
| 3 | アニメーション | 段階的表示が動作 |
| 4 | 画像 | 全画像がロード、WebP最適化 |
| 5 | CTA | 目立つ位置に配置、クリック可能 |
| 6 | パフォーマンス | LCP < 2.5s |
| 7 | AI生成っぽさ | NG集の項目に該当しない |
| 8 | ベンチマーク比較 | 構成が模倣できている |

## 5.3 ユーザーレビュー

スクリーンショットをユーザーに共有し、フィードバックを受ける。
修正は即座に反映。

## Pass 5: 修正ルール

- FAIL項目のみ修正 → 再スクリーンショットで検証
- 2回FAILで該当セクションをPhase 2に戻して再設計
