# Learn from Mistake: $ARGUMENTS

## 1. ミスの分析

- 何が間違っていたか(事実)
- なぜ間違えたか(根本原因)
- どうすれば防げたか(対策)

## 2. ルール策定

- 具体的で検証可能なルール
- 例外ケースの明記
- 簡潔な表現(1-2行)

## 3. 保存先の判定と更新

### CLAUDE.md (開発ルール)
コーディング規約、ツールの使い方、技術的な制約に関するミスの場合:
- `~/.claude/CLAUDE.md` の適切なセクションに追加
- 既存ルールとの重複排除
- 優先度が高ければ `**IMPORTANT**` を付与

### ナレッジベース (ビジネス知見)
事業判断、外部サービスの仕様、ビジネスプロセスに関するミスの場合:
- `~/.claude/knowledge/` の適切なカテゴリに記録
- 日付、コンテキスト、教訓を構造化して保存

### エージェントメモリ (専門知識)
特定エージェントの専門領域に関するミスの場合:
- 該当エージェントのメモリに記録されるよう明示的にメモを残す
- 例: セキュリティ関連 → security-auditor、型関連 → type-checker

### 3b. INDEX.md更新（新規ナレッジファイル作成時のみ）

新規ナレッジファイルを作成した場合のみ、`~/.claude/knowledge/INDEX.md` の適切なクラスタにファイルを追加。
既存ファイルへの追記の場合はINDEX.md更新不要。

## 4. 確認

- 更新後のルールを表示
- ユーザーの承認を得る
- 「このルールで再発を防げるか？」を自問

### 使い方

```
/learn-mistake Supabaseの.single()で0行の時に406エラーが出た
/learn-mistake Stripeの金額をセント単位で送ってしまった(JPYはゼロ小数点)
```
