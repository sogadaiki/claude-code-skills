# 事業計画系プロジェクト用エージェント指示（business-plan）

## BusinessModelAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。全ファイル・全フォルダを走査すること）

以下を調査して報告:

1. ディレクトリ構造: 全フォルダのツリー（深さ3レベルまで）
2. 事業概要の抽出:
   - 事業名、法人名、代表者、事業領域
   - ビジネスモデル（収益構造、顧客セグメント、提供価値）
3. 収支・財務データの特定:
   - 売上・コスト・利益に関する記載があるファイルの特定
   - 主要な財務数字の抽出（金額、期間、前提条件）
4. マイルストーン・スケジュール:
   - 事業計画上のフェーズ・期限・KPI目標

注意: 個人情報（住所、電話番号、口座番号）はサマリに含めない
調査のみ。ファイル編集禁止。
```

## MarketAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。市場・競合に言及する全ファイルを走査すること）

以下を調査して報告:

1. 市場・競合に関する記載:
   - ターゲット市場の定義、市場規模
   - 競合分析、差別化要因
2. マーケティング・営業戦略:
   - 顧客獲得チャネル、CAC、LTV
   - 広告・PR・SNS戦略の記載
3. リスク・課題:
   - 事業計画上で言及されているリスク要因
   - 規制・法的制約の記載
4. 資金調達・投資関連:
   - 資金調達計画（クラウドファンディング、融資、出資等）
   - 投資家向け資料の有無と概要

調査のみ。ファイル編集禁止。
```

## StakeholderAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。関係者・契約に言及する全ファイルを走査すること）

以下を調査して報告:

1. ステークホルダー一覧:
   - 創業者・経営陣、投資家、パートナー、顧客
   - 各ステークホルダーの役割と関係性
2. 契約・合意文書:
   - 契約書、覚書、合意書等の存在と概要
3. ToDo・未完了事項:
   - 「準備中」「検討中」「要確認」等の記述
   - 提出期限がある文書
4. 外部連携:
   - 支援機関、行政、金融機関等との関係

注意: 個人情報（住所、電話番号、口座番号）はサマリに含めない
調査のみ。ファイル編集禁止。
```
