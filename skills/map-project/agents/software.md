# ソフトウェアプロジェクト用エージェント指示

## Architect への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。ディレクトリは全階層、grepは複数パターンで網羅的に検索すること）

以下を網羅的に調査して報告:

1. ファイル構造: src/ 以下の全ファイルツリー（ディレクトリ構造）
2. エントリポイント: メインファイルの全ルート/ページ定義
   - パス、メソッド、ハンドラ関数名、認証方式を表形式で
3. 各ファイルのexport関数: 名前と役割を1行ずつ
4. 設定ファイル: 環境変数（名前と用途のみ、値は絶対に書かない）
5. 依存関係: package.json等の主要依存ライブラリ
6. ビルド/デプロイ: wrangler.toml / vercel.json / Dockerfile等の設定内容

調査のみ。コード編集禁止。
```

## DataModeler への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。ディレクトリは全階層、grepは複数パターンで網羅的に検索すること）

以下を網羅的に調査して報告:

1. データベーステーブル/コレクション:
   - テーブル名、主要カラム、PK/FK関係、件数目安
   - ソースファイル内のクエリ文字列やREST APIパスから推定
2. ビュー/マテリアライズドビュー: 名前と用途
3. RPC/ストアドプロシージャ: 名前、引数、戻り値、用途、呼び出し元
4. 型定義: 主要な interface / struct（APIレスポンス型、DBレコード型）
5. ストレージ: ファイルストレージ、CDN、キャッシュ

grepで supabaseGet, supabasePost, supabaseRpc, fetch, prisma, drizzle 等の
DB操作関数の呼び出し箇所を網羅的に検索すること。
調査のみ。コード編集禁止。
```

## IntegrationAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。ディレクトリは全階層、grepは複数パターンで網羅的に検索すること）

以下を網羅的に調査して報告:

1. 外部API連携: 全ての外部サービス（OpenAI, Google, Stripe, Discord等）
   - 使用モデル/エンドポイント、認証方式、呼び出し箇所ファイル:行番号
2. GAS/Webhook連携: スクリプト一覧、トリガー方式、列マッピング、分岐ロジック
3. 認証方式マトリックス: 全エンドポイント × 認証方式
4. シークレット管理: 環境変数名と用途（値は絶対に記録しない）
5. データフロー: 外部入力→処理→保存→出力をASCII図で
   - 複数フローがある場合はA, B, C...で分離

調査のみ。コード編集禁止。
```
