# 法務・文書系プロジェクト用エージェント指示（legal / documents）

## DocumentStructureAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。全フォルダ・全ファイルを走査すること）

以下を網羅的に調査して報告:

1. ディレクトリ構造: 全フォルダのツリー（深さ3レベルまで）
2. ファイル一覧: 各ファイルの種類（PDF/DOCX/XLSX/その他）とサイズ
3. ファイル命名規則の分析: パターンの特定（例: 甲○号証_○○.pdf）
4. 時系列整理: ファイル名・更新日時から読み取れる時系列順
5. ファイル数サマリ: 種類別のファイル数

注意: ファイルの中身は読まない。構造とメタ情報のみ調査。
調査のみ。ファイル編集禁止。
```

## CaseAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。主要文書は全て目を通すこと）

以下を調査して報告:

1. 主要文書の内容サマリ（表紙・ブリーフ・主張書面・答弁書・準備書面等）:
   - 各文書の主要な主張・内容を3-5行で要約
   - 対象ファイル数の目安: 訴状系+答弁書+準備書面+ブリーフ（通常8-15ファイル程度）
   - PDFが20ファイル超の場合は主張書面系を優先し、証拠書類は番号・表題のみ
2. 基本情報の抽出:
   - 事件番号、裁判所名、当事者名（原告・被告・代理人）
   - 請求内容、訴訟物の価額
3. 証拠番号マップ:
   - 甲号証・乙号証等の一覧（番号、ファイル名、内容概要）
4. 時系列イベント:
   - 文書から読み取れる主要イベントの日付順リスト

注意:
- 全PDFを読むことは禁止。主要文書（訴状、答弁書、準備書面、判決等）のみ対象
- 個人情報（住所、電話番号、メールアドレス）はサマリに含めない
調査のみ。ファイル編集禁止。
```

## RelationshipAnalyst への指示

```
プロジェクト: {対象ディレクトリ}
種別: {判定結果}
**調査の網羅性**: very thorough（見落とし禁止。文書間の参照関係を網羅的に追跡すること）

以下を調査して報告:

1. 文書間の参照関係:
   - 主張書面→証拠の参照関係（「甲○号証」等の引用を追跡）
   - 相手方文書への反論対応関係
2. 関連する外部手続き:
   - 本件以外の手続き（懲戒請求、移送申立、仮処分等）の特定
   - 関連事件番号の抽出
3. ToDoと未完了事項:
   - 提出期限・期日が明記されている事項
   - 「準備中」「検討中」「要確認」等の記述の特定
4. 戦略メモ・方針文書の特定:
   - 戦略、方針、メモ等のファイルの存在確認と内容概要

注意:
- 個人情報（住所、電話番号、メールアドレス）はサマリに含めない
調査のみ。ファイル編集禁止。
```
