# 記述ルール（Phase 3.3 詳細）

## 全種別共通

- **情報の信頼度を3段階で明示**:
  - タグなし = 実文書から直接確認済み
  - `(書面参照)` = 実文書を直接読んではいないが、他の書面中での言及から記載
  - `(推定)` = ファイル名・文脈からAIが推論した情報。準備書面・事業判断の根拠として使用禁止
  - `(要確認)` = 矛盾または疑義があり、人間による確認が必要
- **ソース参照の義務化**: 日付・金額・事件番号は末尾に `[src: ファイル名]` を付記
- 原文の表題をそのまま転記する。AIが要約・意訳しない
- 証拠マップの「内容」列は実ファイル名と一致させる
- **値は書かない**。APIキー、PIN、パスワード、個人の連絡先は絶対に含めない
- **件数は目安**。「約100件」レベルで十分
- **ASCII図は簡潔に**。矢印と箱だけ
- **各セクション30行を目安**。超える場合は分離ファイルにリンク:
  - 例: テーブル数が多い → `{memory}/architecture/data-model.md` に分離
  - テーブル記述は1テーブル3行以内（名前、PK/FK、主要カラム）
  - ファイル構造が大きい → `{memory}/architecture/file-structure.md` に分離
