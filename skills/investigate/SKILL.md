# Investigate: $ARGUMENTS

サブエージェントを使用して調査を実行する。メインコンテキストを温存。

## 調査プロセス

1. **タスク分解**: 調査対象を独立したサブタスクに分割
2. **並列探索**: 各サブタスクをサブエージェント(Explore型)に委譲
   - Glob/Grepで関連ファイルを特定
   - 実装パターンを分析
   - 依存関係を把握
3. **統合**: 結果を収集し、重複を排除して統合
4. **サマリー報告**: メインコンテキストに簡潔な結果のみ返す

## 適用場面

- コードベース探索(ファイル構成、実装パターン)
- 依存関係分析
- ログ・エラー調査
- 複数プロジェクト横断の調査
- 技術的な実現可能性調査

## 使い方

```
/investigate 認証フローの実装箇所
/investigate Stripe連携のエラーハンドリング
/investigate 3プロジェクト横断でSupabase RLSの使い方
```

必ずサブエージェントを使用すること。メインで直接Grep/Globを実行しない。
