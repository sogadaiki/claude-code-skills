# マネージャー 5軸分析フレームワーク

> SKILL.md Step 1.2-1.3 の詳細。マネージャーペルソナでDiscordチームchに分析結果を対話形式で投稿する。

## 1-2. 5軸分析 (2-3メッセージ)

以下の5軸で分析し、結果をDiscordに投稿する:

**軸1: カテゴリ配分**
- 今日の投稿カテゴリ(A/B/C/D)を集計
- STAGES/current.md の目標配分と比較
- 直近7日間の配分傾向

**軸2: 品質ゲート通過率**
- 今日のリトライ回数
- 頻出する指摘パターン

**軸3: エンゲージメント**
- posted/ファイルに`engagement:`セクションがあればデータ使用
- なければ「データ未取得」と明記

**軸4: API制約・プラットフォーム変更**
- エラーログから兆候確認

**軸5: コンテンツ品質の定性評価**
- ペルソナらしさの一貫性
- 特に良かった投稿1本を選出

## Discordメッセージテンプレート

分析結果:
```
[title] Manager: 分析結果
[content] 今日はN本投稿。カテゴリ: A/B/C = X/Y/Z本
品質ゲート全通過。カテゴリBの偏りは先週から改善傾向
良かった投稿: 「{タイトル}」-- 冒頭のフックが効いてた
```

改善ポイント:
```
[title] Manager: 改善ポイント
[content] 朝の短文は安定してきた
課題: 夜のBカテゴリ偏り。{具体的な改善提案}
実験EXP-001(スレッド効果): 現在N本中。まだ結論は早い
```

## 1-3. 週次分析 (--weekly時のみ、追加1-2メッセージ)

- 週間投稿数(日別)
- カテゴリ配分実績 vs 目標
- 来週の戦略調整
- `analytics/weekly/YYYY-WXX.json` に週次データ保存
