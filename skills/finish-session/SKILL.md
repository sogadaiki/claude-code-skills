# Finish Session

セッションを一時中断し、次セッションでスムーズに再開できるようにする。
`/finish-task`（PR作成+Issueクローズ）とは違い、作業は継続中のまま中断する。

## 手順

### 1. 未コミットの変更を確認

```bash
git status
git diff --stat
```

変更がある場合のみコミットする（変更なければスキップ）:
```bash
git add -A && git commit -m "wip: セッション中断時の進捗保存"
```

### 2. restart.md を .gitignore に追加（まだ入っていない場合）

```bash
# ルートの .gitignore に restart.md が含まれていなければ追加
grep -q 'restart.md' .gitignore 2>/dev/null || echo 'restart.md' >> .gitignore
```

### 3. restart.md を更新（履歴追記方式）

プロジェクトルートの `restart.md` に **新しいセッションを先頭に追加** する。
既存の内容がある場合は、最新2セッション分だけ残し、3回目以降の古いエントリは削除する。

ファイル構造:
```markdown
# Restart Notes

## Session: {今日の日付}
Branch: {現在のブランチ名}
Issue: {作業中のIssue番号、あれば}

### やっていたこと
{このセッションで何をやっていたか、1-3行}

### 次にやること
{次のセッションで最初にやるべきこと、箇条書き}

### 注意点
{引き継ぎで知っておくべきこと。なければ省略}

## Session: {前回の日付}
{前回のセッション内容がそのまま残る}
```

**ルール:**
- 新セッションを `## Session:` の見出しで先頭に追加する
- `## Session:` ブロックは最大2つまで保持。3つ目以降は削除する
- restart.md の内容は会話の流れから自動で判断する。ユーザーに「何をしていましたか？」と聞かない

### 4. 作業ログを追記

`<ops-repo-path>/logs/{YYYY-MM-DD}.md` に作業内容を追記する。
**どのリポジトリで実行しても、常にメインopsリポの `logs/` に書き込む。**

ファイルが存在しない場合は新規作成（見出しを先頭に）。存在する場合は末尾に追記。

```markdown
# {YYYY-MM-DD} 作業ログ          ← ファイル新規作成時のみ

---

## {HH:MM} {リポジトリ名}
**ステータス**: 中断（継続中）
**Issue**: #{番号}（あれば）

### やったこと
- {箇条書き、具体的に}

### タグ
{該当するものだけ: #development #design #marketing #content #bugfix #refactor #infra #client}
```

**ルール:**
- セッションの会話から自動で判断する。ユーザーに聞かない
- 「やったこと」は第三者が読んでも分かる粒度で書く
- タグは1-3個。後からgrepで検索できるようにする
- 時刻はログ書き込み時点のローカル時刻（`date +%H:%M`）

### 5. ナレッジ保存チェック

セッション中の会話を振り返り、以下に該当するものがあれば該当するメモリ/ナレッジファイルに追記する。
**ユーザーに聞かずに自己判断で実行する。**

#### 保存すべきもの（1つでも該当すれば保存）

| 種類 | 例 | 保存先 |
|------|-----|--------|
| **新しい事実・設定値** | API仕様、アカウント情報、外部サービスの挙動 | memory/ or knowledge/ の該当ファイル |
| **成果物の詳細** | 作成した原稿の内容要約、設計判断の根拠 | memory/ or knowledge/ の該当ファイル |
| **失敗・教訓** | エラーの原因、やってはいけないこと | 該当ファイル + `/learn-mistake` 相当の記録 |
| **外部サービスの制約** | 規約の解釈、審査基準、料金体系 | knowledge/ の該当ファイル |
| **クライアント固有情報** | 確認事項、決定事項、ステータス変更 | memory/ の該当ファイル |

#### 保存しないもの

- 一時的な作業状態（restart.mdで十分）
- コードの変更内容そのもの（gitで追える）
- 推測・未確定の情報（確定したら保存）

#### 実行手順

1. 上記の表に該当する情報がセッション中にあるか振り返る
2. 該当あり → 適切なファイルに追記（既存セクションの更新 or 新規セクション追加）
3. 該当なし → スキップ（「保存すべきナレッジなし」と報告のみ）
4. 保存した場合 → 完了報告に「ナレッジ保存: {ファイル名} に {内容の1行要約}」を追加

**判断基準**: 「次回このテーマに取り組む時、この情報がなかったら困るか？」 → YESなら保存。

### 6. 完了報告

以下を表示:
- コミットしたかどうか
- restart.md の内容（確認用）
- 作業ログに追記した旨
- ナレッジ保存した内容（あれば）
- 「次回 `/resume` でこの続きから再開できます（朝一番なら `/morning`）」

## 注意

- restart.md は `.gitignore` に入れる（個人の作業メモなのでリポジトリに入れない）
- `/morning` は restart.md を読み込むが **削除しない**（クラッシュ復帰用に最低2セッション分保持）
- PRは作らない、Issueはクローズしない
- ブランチはそのまま（切り替えない）

## コンテキスト制約

- **restart.mdにはこのプロジェクトの情報のみ記録する。** 他リポジトリのIssueやステータスは記載しない
- 例外: メインopsリポでの実行時のみ、複数プロジェクトの状況を「次にやること」に含めてよい
- 各プロダクトリポで実行した場合、restart.mdにはそのリポの作業内容だけを書く
