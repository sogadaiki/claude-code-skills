# Contributing

スキルの改善・新規追加を歓迎します！

## スキルの追加方法

### 1. ディレクトリ構成

```
skills/your-skill-name/
├── SKILL.md              # 必須: メインプロンプト
├── references/            # 任意: 参照ドキュメント
│   └── some-reference.md
├── templates/             # 任意: テンプレートファイル
└── scripts/               # 任意: ヘルパースクリプト
```

### 2. SKILL.md のフォーマット

```yaml
---
name: your-skill-name
description: スキルの説明（1行で簡潔に）
argument-hint: [引数の説明]
---

# スキルタイトル: $ARGUMENTS

## やること

1. ステップ1
2. ステップ2
...
```

### 3. 公開前チェックリスト

- [ ] `/audit-skill your-skill-name` でセキュリティ監査を実行
- [ ] 個人パス (`/Users/...`) が含まれていない
- [ ] APIキー・トークンがハードコードされていない
- [ ] クライアント名・個人名が含まれていない
- [ ] 内部URLが含まれていない
- [ ] スキル単体で動作する（or 依存関係が明記されている）

### 4. PR テンプレート

```markdown
## スキル名
[name]

## 何ができるか
[1-2文で説明]

## 使い方
[コマンド例]

## 依存関係
[必要な外部ツール・設定があれば]
```

## スキル改善のガイドライン

- **ロジックを壊さない**: 既存スキルの改善時は、元の意図を維持する
- **日本語OK**: 日本語でのPRを歓迎します
- **テスト**: 可能であれば、スキルを実際に実行して動作確認してからPR
- **過度な抽象化を避ける**: シンプルで理解しやすいプロンプトが良いスキル

## 設計パターンのヒント

良いスキルに共通する特徴:

1. **明確なフェーズ分け** — 入力→処理→出力が構造化されている
2. **品質ゲート** — 生成物の品質基準が明示されている
3. **フォールバック** — うまくいかない時の対処が書かれている
4. **使い方の例** — 末尾に具体的なコマンド例がある
